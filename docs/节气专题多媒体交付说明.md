# 二十四节气网站 · 节气专题多媒体交付说明

> **推荐阅读：** 全站各页面「图片 / 文字 / 音频」替换位置的总览，见 **[内容替换指南.md](./内容替换指南.md)**。  
> 本文档侧重节气专题页的媒体命名与实现细节。

---

## 一、整体思路

节气专题页采用 **「数据驱动 + 统一模板」** 的方式：

1. 所有节气的文案、图片、音频、视频路径集中在 **`js/term-pages-data.js`** 中配置；
2. 页面打开后，由 **`js/term-page.js`** 读取当前节气数据，动态生成 HTML；
3. 视频播放由 **`js/video.js`** 初始化；音频点击播放同样复用该脚本；
4. **24 个节气**均有独立 HTML（`html/terms/{slug}.html`），由 `scripts/generate-term-html.js` 生成；`html/term.html?t=slug` 会跳转到对应页。

这样你只需 **改数据文件 + 往文件夹里放媒体**，不必逐个改 HTML。

## 二、单个节气页的结构

从上到下依次为：

| 顺序 | 区域 | 锚点 | 内容 |
|------|------|------|------|
| 1 | 顶部导航 | — | 全站 6 栏导航（Bootstrap Navbar） |
| 2 | 节气横条 | — | 24 节气快捷切换（`js/term-nav.js` 生成） |
| 3 | Hero | — | 节气名、日期、简介 |
| 4 | 诗词书画 | `#poetry` | 诗句、文献、书画说明 |
| 5 | 民俗美食 | `#food` | 食俗、做法、配图（暖黄色调） |
| 6 | **节令音频** | `#audio` | **1 段可点击播放的音频** |
| 7 | 节气影像 | `#media` | **2 个并排视频播放器** |
| 8 | 页脚 | — | 版权与链接 |

背景动画（雨、梅花、春雷、谷雨密集降水、清明燕鸟等）由 **`js/term-fx.js`** 根据 `body` 上的 `data-fx` 属性加载；立春 / 雨水 / 清明 / 谷雨有专属动效脚本。

## 三、媒体文件放哪里、怎么命名

### 3.1 目录

```
HTML/
├── images/terms/          ← 节气配图（SVG/JPG/PNG）
├── media/terms/           ← 节气视频 + 音频
│   ├── lichun-1.mp4
│   ├── lichun-2.mp4
│   ├── lichun.mp3
│   ├── yushui-1.mp4
│   └── …
└── js/
    ├── term-slugs.js      ← 节气中文名 → 拼音 slug
    └── term-pages-data.js ← 各节气内容与媒体配置
```

### 3.2 命名规则

| 类型 | 数量 | 命名 | 示例（立春） |
|------|------|------|--------------|
| 视频 | 每节气 **2 个** | `slug-1.mp4`、`slug-2.mp4` | `lichun-1.mp4`、`lichun-2.mp4` |
| 音频 | 每节气 **1 个** | `slug.mp3` | `lichun.mp3` |
| 图片 | 按需 | 文件名写在 data 里 | `caitai.svg` |

`slug` 与 `js/term-slugs.js` 中一致，例如：立春 `lichun`、雨水 `yushui`、清明 `qingming`。

### 3.3 格式建议

- **视频**：H.264 编码的 `.mp4`，720p 或 1080p，单文件约 50MB 以内  
- **音频**：`.mp3`，单文件约 10MB 以内  
- **图片**：`.jpg` / `.webp` / `.svg`，单张约 500KB 以内为宜

## 四、音频功能是怎么做的

### 4.1 数据配置（`term-pages-data.js`）

每个节气增加 `audio` 字段，例如立春：

```javascript
audio: {
  title: '立春 · 东风解冻',
  subtitle: '古琴与箫合奏',
  desc: '以清越琴音迎接阳气初生，呼应立春「万物起始」之意。',
  src: 'lichun.mp3'
}
```

- `title` / `subtitle`：播放器上显示的标题与副标题  
- `desc`：区域说明文字  
- `src`：文件名（相对 `media/terms/`）

其余 20 个节气在文件末尾的自动生成逻辑里已统一配置为：

```javascript
audio: {
  title: t.name + ' · 节令音频',
  subtitle: '节气主题音乐 / 自然声景',
  desc: '聆听' + t.name + '时节的声音记忆，感受节令流转。',
  src: slug + '.mp3'
}
```

### 4.2 页面渲染（`term-page.js`）

在渲染完美食区、视频区之间，向 `#audioZone` 插入：

- 一块 **`.audio-card`** 卡片（与「节气知识」页音频样式一致）  
- `data-audio` 属性指向 `../../media/terms/xxx.mp3`（或 `../media/terms/`）  
- 内含播放图标、标题、进度条  

### 4.3 交互逻辑（`video.js`）

- 将音频点击事件改为 **事件委托**：`$(document).on('click', '.audio-card', …)`  
- 这样 **动态插入** 的音频卡片也能正常播放  
- 点击卡片：播放/暂停，同时更新进度条；切换另一段音频时会停止上一段  

### 4.4 样式（`css/term-pages.css`）

新增 **`.term-zone-audio`** 区域：浅蓝绿背景，与视频区、美食区区分开。

## 五、视频功能是怎么做的（每节气 2 个）

### 5.1 数据配置

每个节气使用 **`videos` 数组**（2 项）：

```javascript
videos: [
  { title: '雨水 · 烟雨江南', desc: '…', src: 'yushui-1.mp4' },
  { title: '雨水 · 挖荠包馄饨', desc: '…', src: 'yushui-2.mp4' }
]
```

### 5.2 页面渲染

- `term-page.js` 用 Bootstrap 栅格 **`col-lg-6`** 并排两个播放器  
- 每个播放器包含：`<video>` + 播放/静音/进度条/时间  

### 5.3 多播放器初始化（`video.js`）

- `initFilmVideoPlayer($root)` 会遍历 `$root` 内所有 **`.film-player`**  
- 每个播放器独立绑定控件，互不干扰  
- 节气页渲染完成后调用：`initFilmVideoPlayer($('#mediaZone'))`

## 六、图片功能是怎么做的

- 图片路径配置在 `term-pages-data.js` → `food.images[]`  
- 仅写文件名，如 `caitai.svg`  
- `term-page.js` 根据页面路径自动拼接：  
  - 专题页：`../../images/terms/`  
  - 通用页：`../images/terms/`  
- 每张图下方有 **「配图作用」** 说明文字

## 八、你如何替换自己的素材

### 步骤 1：放入文件

以 **雨水** 为例，将以下文件复制到 `media/terms/`：

```
yushui-1.mp4    ← 视频一
yushui-2.mp4    ← 视频二
yushui.mp3      ← 音频
```

图片放到 `images/terms/`，并在 data 里改文件名。

### 步骤 2：（可选）改标题与说明

编辑 `js/term-pages-data.js` 中 `yushui` 的 `videos`、`audio`、`food.images`。

### 步骤 3：预览

浏览器打开：

```
html/terms/yushui.html
```

- 滚到 **AUDIO** 区域点击卡片播放音频  
- 滚到 **VIDEO** 区域播放两个视频

## 十、技术栈说明（与课程要求相关）

- **HTML5**：`<video>`、`<audio>` 原生标签  
- **CSS3**：专题页动画、蓝绿/暖黄分区色调  
- **JavaScript + jQuery**：数据渲染、播放器、导航、特效  
- **Bootstrap 5**（本地 `lib/`）：栅格布局、导航栏、部分组件  
- 无 Vue/React，无 CDN 外链
